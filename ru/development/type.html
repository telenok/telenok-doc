Создание "Типа объекта"

В Вашем приложении может быть огромное количество объектов самых разных, начиня с Новостей и заканчивая Валютой или Городом.

При создании нового типа объекта ядро приложения создает в папке путь-к-приложению/app/Model/ класс модели, которым Вы можете оперировать далее.

Так же в Базе данных создается таблица с названием, соответствующем полю "code", как показано в примере ниже. Так же автоматически создаются несколько обязательных полей, таких как Название, Активность, Автор создания, Автор обновления, Автор блокировки и Доступ.

Прмер создания нового типа объекта

    
    (new \App\Telenok\Core\Model\Object\Type())->storeOrUpdate([
        'title' => ['en' => 'License'],
        'title_list' => ['en' => 'License\'s list'],
        'code' => 'license',
        'class_model' => '\\App\\Model\\License',
        'active' => 1,
        'multilanguage' => 1,
    ], true, false);

В поле "code" задается название-ключ объекта в контексте всего приложения. Значение из этого поля используется для генерации новой таблицы в Базе данных. Необходимо заметить, что Вы никогда не должны менять значение поля "code" после создания типа, иначе приложение начнет работать с ошибками.

В поле "class_model" задается имя класса, желательно, совпадающего с полем code. В случае, если поле пустое - класс создастся автоматически с именем, соответствующем полю "code". При этом название будет преобразовано по правилу CamelCase. Например, my_model преобразуется в MyModel.

В поле "active" задается активность нового типа. Если поле "active" равно 0, то тип, как и любой другой элемент приложения, создается в черновике и не является активным, то есть не должен использоваться для манипуляции данными.

В поле "multilanguage" задается мультиязычность поля "Название", о котором шла речь выше и которое создается автоматически при создании типа.

Все модели приложения наследуются от класса \App\Telenok\Core\Interfaces\Eloquent\Object\Model, содержащего метод storeOrUpdate(). В метод storeOrUpdate() передаются три параметра

    - массив или объект коллекции \Illuminate\Support\Collection c полями "код поля"-"значние поля", например ['title' => ['en' => 'John']]
    - булевое значение - в случае true в методе проверятся все права на операцию и на модификацию полей
    - булевое значение - в случае true будет создано сообщения с аргументов в виде этой записи для промежуточной модификации полей записи в Вашем коде
